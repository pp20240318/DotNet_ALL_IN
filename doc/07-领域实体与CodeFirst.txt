【核心概念】
================================================================================

  Code First：
    _db.CodeFirst.InitTables(typeof(Admin), typeof(Brand), ...) 启动时自动建表。

  雪花 ID：
    SnowFlakeSingle.Instance.NextId() 生成全局唯一 long 型主键，适合分布式场景。

  BaseEntity 继承：
    所有业务实体继承 BaseEntity，统一 Id、软删除、乐观锁、审计字段。

  表名映射：
    [SugarTable("brand")] 指定数据库表名（小写蛇形）。

================================================================================
【常见问题】
================================================================================

  Q: 表已存在但结构变了怎么办？
  A: SqlSugar Code First 不会自动删列；开发环境可 DROP TABLE 后重启，或手动 ALTER。

  Q: 为什么 Id 用 long 不用 int？
  A: 雪花 ID 为 64 位 long，避免自增 ID 在分布式/分库分表时冲突。

  Q: ProductStatus 默认值是什么？
  A: 实体默认 Draft；种子商品设为 OnSale（上架）。

  Q: 种子数据重复插入？
  A: DbInitializer 先检测 brand 表是否有记录，有则跳过 SeedCatalogData。
